\section{Class Declarations}

In \tqfullvref{sec:ClassAndInterfaceDeclarations} we already described the general structure of declarations for \Java classes, interfaces, enums, records, and annotations, while the use of blanks, round brackets, curly braces and linefeeds was discussed in \tqfullvref{sec:WhiteSpace}.

Based on that, the declaration for a class, interface or enum starts with the respective Javadoc comment for the type, followed by the annotations for it, if any, and the line setting the name for the type, as well as the base class and implemented interfaces.

If the type is sealed\index{sealed}, the \lstinline|permits|\index{permits} clause is placed on the following line, with the default indentation, before finally the line with the opening curly brace follows.

Basically the declaration for a record looks the same, but because it declares the attributes also on that first line, that could be longer and may require to be wrapped.

Each type ends with a comment after the closing curly brace, followed by a comment line repeating the name and kind of the type.

Some samples\footnote{I omitted the Javadoc comments from the samples for brevity.}:
\begin{lstlisting}
public final class MySimpleClass
{
    …
}
//  class MySimpleClass
\end{lstlisting}

\index{java.lang!CharSequence}
\begin{lstlisting}
public interface MyInterface<T extends CharSequence>
{
    …
}
//  interface MyInterface
\end{lstlisting}
 
\begin{lstlisting}
public enum MyEnum
{
    …
}
//  enum MyEnum
\end{lstlisting}

\index{java.lang!String}
\begin{lstlisting}
public final class MyRecord( String name, int age, boolean retirementStatus )
{}
//  record MyRecord
\end{lstlisting}
 
\index{java.lang!Serializable} 
\begin{lstlisting}
public final class MyClass extends MyBaseClass implements MySealedInterface, Serializable
{
    …
}
//  class MyClass
\end{lstlisting}

\index{java.lang!String}\index{java.util!List}
\begin{lstlisting}
public sealed class MySealedInterface extends List<String>
    permits MyClass
{
    …
}
//  interface MySealedInterface
\end{lstlisting}

\index{java.lang!FunctionalInterface}
\begin{lstlisting}
@FunctionalInterface
public interface MyFunctionalInterface<T>
{
    …
}
//  interface MyFunctionalInterface
\end{lstlisting}

\index{java.lang.annotation!Annotation}\index{java.lang.annotation!Documented}\index{java.lang.annotation!Retention}\index{java.lang.annotation!Target}\index{java.lang.annotation!RetentionPolicy!RUNTIME}\index{java.lang.annotation!ElementType}
\index{java.lang.annotation!ElementType!ANNOTATION\_TYPE}
\begin{lstlisting}
@Documented
@Retention(RUNTIME)
@Target(ANNOTATION_TYPE)
public @interface MyAnnotation
{}
//  @interface MyAnnotation
\end{lstlisting}

Inner classes\footnote{…, interfaces, enums, records, and annotations – although the latter are rarely defined as inner classes} will follow the same pattern, but with an additional level of indentation. Inner classes are discussed in detail in chapter \tqvref{sec:InnerClasses}.

The elements of the class (inner classes, constants, attributes, constructors, methods, …) are all separated from each other by a single blank line (see chapter \tqfullvref{sec:BlankLines}.

\subsubsection{Principles}
\begin{enumerate}[label=P\arabic*.]
\item{Only one public top-level class (interface, record, enum, …) per compilation unit. Avoid non-public top-level classes.}
\item{Group the parts of the class as shown in \tqvref{sec:ClassAndInterfaceDeclarations}. Remove the comments for unused sections.}
\item{Place each class annotation on a line of its own.}
\item{The \lstinline|permits|\index{permits} gets a line of its own, too.}
\item{No blank before the opening angle bracket for a type variable. No blanks after a comma inside the angle brackets.}
\item{The opening curly brace is again on a line of its own.}
\item{The closing curly brace is followed by a line with a comment repeating type and name of the class.}
\item{A single blank line separates the elements of a class (see chapter \tqfullref{sec:BlankLines}.}
\end{enumerate}
