\section{‘return’ and ‘yield’ Statements}
The \lstinline|return|\index{return} statement has to be the last statement of a method, while the \lstinline|yield|\index{yield} statement has to be the last statement of a \lstinline|case| block; refer to the chapter \tqfullvref{sec:ReturningValues} for more details on this.

When \lstinline|return| is always the last statement of a method, it means that the simple form
\begin{lstlisting}
return;
\end{lstlisting}
is obsolete and will be omitted. This implies that there is only one \lstinline|return| statement per method allowed, and that short-cutting a method is a forbidden practice. 

\lstinline|yield|\index{yield} alone (without a return value) in a \lstinline|case| block will always cause a compiler error. Some more details can be found at \autocite{ORACLE_DOC_LANGUAGE_SPECIFICATION:Yield}.

The use of parentheses with the \lstinline|return|\index{return} or \lstinline|yield|\index{yield} statement is not required unless they make the return value more obvious and better readable in some way.

The recommended forms are shown in lines~15 and 22/23, below:
\index{java.util!Arrays!stream()}\index{org.tquadrat.foundation.exception!NullArgumentException}\index{org.tquadrat.foundation.utils!StringUtils!abbreviate()}\index{java.lang!String!isBlank()}\index{java.lang!String!startsWith()}\index{java.lang!String!length()}\index{java.util.stream!Stream!filter()}\index{java.util.stream!Stream!collect()}\index{java.util.stream!Collectors!joining()}\index{org.tquadrat.foundation.exception!UnsupportedEnumError}
\begin{lstlisting}[numbers=left]
public final String myMethod( final InputType inputType, final String... input )
{
    InputSwitch: final var retValue = switch( inputType )
    {
        case null -> throw new NullArgumentException( "inputType" );
        case FILE -> 
        {
            var result = stream( input )
                .filter( String::isBlank )
                .filter( s -> !s.startsWith( " " )
                .collect( joining( "\n" );
            result = result.length() > 80
                ? abbreviate( result, 80 )
                : result;
            yield result;    
        }
        case REGULAR -> abbreviate( input [0], 80 );
        case UNKNOWN -> "Unknown input";
        default -> throw new UnsupportedEnumError( inputType );
    }   //  InputSwitch:
    
    //---* Done *----------------------------------------------------
    return retValue;
}   //  myMethod()
\end{lstlisting}
